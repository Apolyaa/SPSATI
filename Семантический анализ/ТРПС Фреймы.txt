    
    Методы систем искусственного интеллекта.
    
    Наибольшее распространение получили следующие модели представления знаний : 
    - продукционная модель или представление знаний с помощью правил; 
    - фреймовая модель;
    - модель семантических сетей.
    Фреймовая модель
    Фрейм - это знание, полученное на основании факта, которым является имя фрейма.  
    Фрейм содержит подструктуры, называемые "слотами". Слоты  определяют, имеется ли в конкретной ситуации  факт, который представляет имя фрейма.
    Слоты имеют свое имя и значение. 
      
    где F - знак фрейма;  - предикат "быть", "являться"; 
    N - имя фрейма;  - имя слота;  - значение слота;  - имя процедуры.
    Значениями слота могут быть: 
    * числа, математические соотношения, тексты на естественном языке; 
    * программы, правила вывода или ссылки на другие слоты данного фрейма или других фреймов (Связи между фреймами задаются значениями специального слота с именем "связь". В итоге реализуется принцип «матрешки»);
    * слоты могут содержать присоединительные процедуры, позволяющие производить некоторые операции для получения значения этого слота. 
    * слоты могут быть и незаполненными (пустыми). Они могут заполняться в процессе активизации фрейма в соответствии с определенными условиями. 
    Вид связей между фреймами на сети фреймов может быть произвольным. Это определяется целью, поставленной перед системой для решения конкретной задачи.
    В общем виде тип связи имеет характер "нелинейности" переходов, т.е. подобен "гипертексту".
    Значениями слотов могут быть имена систем, в памяти которых хранится разнородная SI (всех форм представления, характерная для естественного языка). В этом случае речь идет о "мультимедиа" фреймах.
    
    Во фреймовых моделях выделяют две части : 
    * набор фреймов, образующих библиотеку внутреннего представления знаний;
    * механизм их преобразования или связывания между собой.
    Фреймы представляют собой декларативно-процедурные структуры, т.е. совокупность описаний и в ряде случаев связанных с ними процедур, доступ к которым выполняется прямо из фрейма.
    Рассмотрим подробнее структуру данных фрейма (см. рис).
    
      	Имя фрейма
      			Указатель наследования
      				Указатель атрибутов слота
    	Имя фрейма
Слот 1Слот 2. . . . .Слот n    
     Имя слота			Значение слота	Демон
    
    
    Имя фрейма - идентификатор, уникальное имя фрейма в данной фреймовой системе.
    Имя слота - уникальное имя во фрейме, к которому он принадлежит. Отображают структуру конкретного знания, которое имеет имя совпадающее с именем фрейма. Некоторые слоты могут быть системными (служебными). Например, слот IS-A показывает фрейм -родитель данного фрейма.
    Указатели наследования - используются только во фреймовых системах иерархического типа, основанных на отношениях «абстрактное -конкретное». Они показывают, какую информацию об атрибутах слотов во фрейме верхнего уровня наследуют слоты с такими же именами во фрейме нижнего уровня.
    Указатель атрибутов слота  - указывает тип значения слота (текст, число, процедура, указатель и т.п.) 
    Значение слота - непосредственно значение в соответствии с типом.
    
    Пример декларативного фрейма:
    «Хорошая квартира»			
Имя слотаЗначение Этаж2-4 Кухня>10 кв.м. Комнаты>15 кв.м. СанузелРаздельный ПолПаркет Потолок> 2.7 м ДомЭлитный    
    Если значения слотов определены (конкретизированы), то можно считать, что имеет место факт «Хорошая квартира». 
    
    Пример процедурного фрейма:  Пусть цель – купить квартиру.
    
    «Покупаем квартиру»
Имя слотаЗначениеКвартираХорошаяРайонПрестижныйИнфраструктураРазвитаяЦенаУстраивает    
    Если данный фрейм определен, то мы принимаем решение купить квартиру.
    Конкретизация 1-го слота сводится к конкретизации фрейма «Хорошая квартира».
    Конкретизация 2-го слота сводится к конкретизации фрейма:

Метафрейм – определяет стратегию вывода. 




Механизм логического вывода применительно к фреймам
    
    МЛВ – это модуль интеллектуальной системы, в котором реализованы методы обработки знаний, позволяющие на основе известных знаний  получать новые знания способствующие достижению цели системы.
    
    В процессе получения вывода возникают задачи: 
    какой фрейм или какой слот считать означенным (конкретизированным).
    
    Слот считается конкретизированным, если его значение определено. Если в качестве значения слота выступает другой фрейм более низкого уровня , то конкретизация слота сводится к означиванию этого фрейма.
    
    Рассмотрим способы означивания фрейма.
    1.  «Означивание по конъюнкции слотов».
                «Зачет получен»
Имя слотаЗначениеРубежный контроль 1 Рубежный контроль 2Рубежный контроль 3Домашнее заданиеЛабораторные работы    
    
    2. «Означивание по конъюнкции группы слотов».
            «Абитуриент зачислен»
ИмяЗначениеНаучный тур*Академический тур*Документы * #ЕГЭ    #Общий конкурс   #    
    3. «Означивание по информативному слоту».
            «Сегодня плохая погода»
Имя слотаЗначениеТемпература-40ОсадкиВетерВлажностьДавление    
    4. «Означивание по порогу». Может быть задан вес для каждого слота. Если суммарный вес превышает заданный порог, то фрейм конкретизирован.
              «Человек болен ОРЗ»
Имя слотаЗначениеВесТемператураВысокая50ДавлениеВыше нормы20Голова Болит20ГорлоБолит30НасморкСильный20    
 
    Выделяют следующие способы управления выводом применительно к фреймам:  
    - с помощью присоединительных процедур (демона и служебной процедуры) ;
    - с помощью механизма наследования.
    
    Демон - это процедура, связанная со слотом фрейма и автоматически запускаемая при обращении к слоту. 
    Отличие демона от служебной процедуры в том, что он конкретизирует слот без обращения к другим фреймам. 
    Демон дополняет фрейм, он может являться запросом к пользователю, т.е. непосредственно быть связанным с интерфейсной частью экспертной системы.
    Служебная процедура - это процедура, которая позволяет конкретизировать слот на основе означенного фрейма более низкого уровня.
